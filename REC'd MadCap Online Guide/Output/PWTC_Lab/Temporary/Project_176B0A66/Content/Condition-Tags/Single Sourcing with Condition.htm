Single-Sourcing with Condition Tags

Condition tags are one of the MadCap Flare’s most prominent single-sourcing features. Creating a condition tag allows you to add or exclude content depending on the output. For example, you may want to create a condition tag for content that you only want to appear in print versions, and another for online versions.

 

Adding a Condition Tag Set

  1. Open the Project Organizer.
  2. Right-Click the Conditional Text folder. A drop down menu appears.
  3. Select Add Condition Tag Set.
  4. Double-Click the condition tag set to rename it.

 

Creating a Condition Tag

  1. Open the Project Organizer.
  2. Double-click on the Conditional Text folder. The folder opens and displays your existing condition tag sets.
  3. Double-click on your condition tag set. The Condition Tag Set Editor opens in the [big window]. Flare provides two default condition tags: a PrintOnly tag for print outputs, and a ScreenOnly tag for online outputs.
  4. Click the New Item button in the local toolbar of the Condition Tag Set Editor.

 

 

You’re not done yet! Condition tags will not be reflected in your output unless you apply them to your content.

 

 

Applying a Condition Tag to a Small Text Selection

  1. Open the Content Explorer.
  2. Double-click on your content file.
  3. Click and drag your cursor in the XML Editor to highlight text within a paragraph to which you want to apply the condition tag.
  4. Open the Project Organizer with your text still selected.
  5. Click on the arrow to the left of the Conditional Text folder to open it. The Conditional Text dialog appears.
  6. Click on the arrow to the left of the conditional text set to expand it.
  7. Click and hold the condition tag, then drag it to the selected text in the XML Editor.
  8. Release the cursor to apply the condition tag to the selected text.

 

Applying a Condition Tag to a Large Text Selection

  1. Open the Content Explorer.
  2. Double-click on your content file.
  3. Click and drag your cursor in the XML Editor to highlight the block of text to which you want to apply the condition tag.
  4. Open the Project Organizer with your text still selected.
  5. Click on the arrow to the left of the Conditional Text folder to open it. The Conditional Text dialog appears.
  6. Click on the arrow to the left of the conditional text set to expand it.
  7. Click and hold the condition tag, then drag it to the structure bar to the left of the selected text.
  8. Release the cursor to apply the condition tag to the selected text.

 

 

Now that you’ve applied condition tags to selected content, the last step is to tell Flare which content to include and exclude in each Target.

 

 

Associating Condition Tags with Targets

  1. Open the Project Organizer.
  2. Double-click the Targets folder. The folder opens and lists your project targets.
  3. Double click your target. The Target Editor opens in the Editor Window.
  4. Double-click the Conditional Text tab on the left hand side of the Target Editor.
  5. Check the boxes to determine which tags to include in your output. By default, condition tags that are not checked to be included will automatically be excluded from that particular output.
